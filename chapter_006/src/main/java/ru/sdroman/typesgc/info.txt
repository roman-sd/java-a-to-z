- Serial GC работает стабильно, но быстро "забивавет память" кучи. Подходит для одноядерного процессора.
- Parallel GC хорошо отрабатывает, но требует большую память кучи.
- Concurrency Mark Sweep GC на моем примере существенных отличий от Parallel GC нет, но на "большой" программе
время Stop the world будет меньше
- G1 GC показал хороший результат, держит кучу в чистоте при относительно небольших затратах памяти.

Во всех тестах для наглядности использовал плагин VisualGC от VisualVM
